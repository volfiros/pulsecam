## Commits

- Write commit messages plain and human — short, direct, in the user's voice.
- Follow the existing conventional-commit style: `type: lowercase short message` (types seen: `feat`, `fix`, `chore`, `docs`, `perf`). No body unless truly needed.
- Do **not** add a `Co-Authored-By:` trailer.
- Do **not** add a "Generated with Claude Code" footer or any similar attribution.

## Code style

- Do **not** add comments unless absolutely necessary (non-obvious *why*, hidden constraint, workaround). Well-named identifiers should carry the meaning.
- No docstrings, banner comments, or "what this does" narration.
